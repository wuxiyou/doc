#### 利用 nginx 的反向代理 搭建负载均衡
~~~
* 情景： 负载均衡 就相当于 是 多人同时干一件事情。比如：100个客户买单 如果只有一个收银台的，那就需要慢慢去排队买单
         耗时可能30分钟。但如果有两个收银台的话 就可以将100个人分为两组，每组50人，这样耗时就可以降低一半(15分钟)
         
         负载均衡的目的：提高客户端的响应时间，降低单台服务器的处理压力，可以做到分流的作用
~~~

#### 实例
~~~
1. 找到 nginx.conf 这个文件  并编辑

在 gzip on 后一行加入 类似
upstream www.rare.com {
    server 192.168.1.2:80 weight = 10;
    server 192.168.1.3:80 weight = 10;
}

www.rare.com  是跟自己取的一个域名，只要好记就可以了

192.168.1.2 跟 192.168.1.3 是服务器的ip地址，采用的是轮询模式，权重为 10

~~~
